# 冒泡排序法

# 算法原理

假设有一个大小为 n 的乱序数组

  • 从索引 0 开始依次比较相邻的两个元素，直到数组尾。
  • 如果前一位元素比后一位元素大的话，交换这两个元素的位置。
  • 每一次循环，都可以得到最后一个数是该数组的最大值。
  • 每次循环忽略上一次循环的最后一位元素。
  • 直到所有元素都完成排序。

# 复杂度分析

  • 时间复杂度：当是最好情况时（只进行了一次循环，就排序完成），时间复杂度为 O (n), 最坏情况则是要进行 (n*n)/2，时间复杂度为 O (n2)，平均时间复杂度为 O (n2)。
  • 空间复杂度：O (1)。

# 选择排序法

# 算法原理

  • 从数组索引 0 位置开始，到数组尾，找到数组中最小的数字，记录它的索引。
  • 交换数组的第一个和最小的数字。
  • 每次循环起始位置加 1，交换起始位置和最小数字。
  • 直到起始位置为数组尾时退出循环。

时间复杂度：不管原数组顺序如何，该排序方法都要进行 (n*n)/2，所以该方法的时间复杂度：O (n2)，平均时间复杂度为 O (n2)。

空间复杂度：O (1)。

# 插入排序法

# 算法原理

  • 从数组头开始依次往后循环
  • 从第二个数开始，依次与前一个数进行比较
  • 如果该数字比前一个数大则继续，反之则与前一个数交换，并且从这个数开始依次往前循环，直到数组头退出
  • 循环到数组尾结束排序

时间复杂度：如果数组原本是一个有序数组，也就是说最小的时间复杂度为 O (n)，如果该数组是最坏的情况，那么时间复杂度为 O (n2)，平均的时间复杂度为 O (n2)。

空间复杂度：空间复杂度为 O (1)。

# 希尔排序法

# 算法原理

  • 取增量 k（数组长度 / 2），从数组头开始，比较该索引和该索引 + k 的两个数，如果该索引的数较大就进行交换
  • 索引值 + k 等于数组大小进入下一次循环
  • 每一次循环 k 的值取上一次循环的 k 值的一半

# 复杂度分析

时间复杂度：他的最好的情况与最坏的情况和插入排序是一样的，但是因为它会优先比较距离比较远的元素，所以他的平均时间复杂度为 O (n1.3)。

空间复杂度：O (1)。

# 归并排序法

# 算法原理

  • 把数组对半分开，直至所有数组都只有单个元素
  • 第一个循环，比较相邻的两个元素，把小的元素放在前面，然后合并成一个数组
  • 每次循环，比较相邻数组的第一个元素，把小的元素放到合并数组的末尾
  • 直到数组合并成原数组大小，退出循环

空间复杂度：O (n)

# 快速排序法

# 算法原理

  • 先用两个指针分别指向数组的开头 (left) 和结尾 (right)，记录数组的第一个数 (temp)，也就是 left 上的数字
  • 比较 right 指针和 left 指针指向的数字
  • 如果 right 的值小于 temp 的值，right 上的数覆盖 left 上的数，右移 left 指针，直到 left 上指向的数大于 temp，把 left 上的值覆盖到 right 上，循环直到 left 等于 right
  • 如果 right 的值大于 temp 的值，左移 right 指针直到 right 上的值大于 temp，覆盖 left 上的数，循环到 left 等于 right
  • 每一次循环，只能确定 temp 值的索引左侧永远小于 temp，右侧永远大于 temp，把数组按 temp 的索引分为左右两个数组
  • 递归，分别排序左数组和右数组，直到左数组和右数组大小为 1 时完成排序

时间复杂度：最好：O (nlog2n)，最坏：O (n2)，平均：O (nlog2n)。

空间复杂度：O (nlog2n)。

# 堆排序法

# 算法原理

  • 把数组想像成一个二叉树
  • 遍历有子树的节点（数组长度一半以内），比较该节点和他的子树，把最小的值放在该节点，是的该节点比他的所有子树都小
  • 遍历结束后，把数组的第一个数（根节点）放入到新的数组中
  • 数排序、得到根节点进行循环
  • 循环原数组的长度的次数，返回的新数组就是排序结束的数组

空间复杂度：O (1)

# 计数排序法

# 算法原理

  • 创建一个数组中最大值大小的计数数组
  • 遍历待排序数组，把待排序数组中的值作为计数数组的索引，计数数组中对应的值加加
  • 遍历计数数组，如果数组中的值大于 1，则在结果数组中推入计数数组的索引
  • 返回结果数组

# 代码实现

# 复杂度分析

时间复杂度：O (n+k)。（k 为计数数组的大小）

空间复杂度：O (n+k)。

# 桶排序法

# 算法原理

  • 设置一定数量的桶，每个桶设置一个大小范围
  • 遍历待排序数组，把里面的元素按大小放入不同的桶中
  • 按顺序遍历桶，对桶中的元素进行排序，然后按顺序推入结果中

空间复杂度：O (n+k)。

# 基数排序法

# 算法原理

  • 创建一个大小为 10 的基数数组
  • 获取待排序数组中数的个位数，然后把这个数作为索引，放入基数数组中
  • 然后从基数数组中按顺序放回原数组中
  • 进行循环，直至数的所有位数都已经被取到
